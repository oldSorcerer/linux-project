
### Настройка перенаправления HTTP → HTTPS

Настройте автоматическое перенаправление всех запросов с HTTP на HTTPS.
- Для Nginx:
  Добавьте в конфигурацию виртуального хоста:
```nginx
server {
listen 80;
server_name example.com www.example.com;

return 301 https://$host$request_uri;
}
```
Перезагрузите Nginx:
```bash
sudo systemctl restart nginx
```
- Для Apache:
  Добавьте в конфигурацию виртуального хоста:
```apache
<VirtualHost *:80>
ServerName example.com
Redirect permanent / https://example.com/
</VirtualHost>
```
Перезагрузите Apache:
```bash
sudo systemctl restart apache2
```
Сайт должен автоматически перенаправлять пользователей на HTTPS-версию при доступе через HTTP.

Требования:
1. Необходимо внести изменения в конфигурационные файлы виртуальных хостов веб-сервера (Nginx или Apache) для настройки перенаправления HTTP на HTTPS.
2. Для Nginx требуется добавить блок `server` с настройками, который слушает порт 80 и перенаправляет запросы с HTTP на HTTPS с использованием кода возврата 301.
3. Для Apache необходимо добавить блок `<VirtualHost>` с командой `Redirect permanent` для перенаправления всех запросов с HTTP на HTTPS.
4. После внесения изменений в конфигурационные файлы требуется перезагрузить веб-сервер (команда `sudo systemctl restart nginx` для Nginx или `sudo systemctl restart apache2` для Apache).
5. Сайт должен автоматически перенаправлять все запросы, поступающие через HTTP, на HTTPS-эквивалент адреса, обеспечивая безопасное соединение.
6. Конфигурация должна учитывать доменные имена (например, `example.com` и `www.example.com`) для обеспечения корректного перенаправления.
7. Перенаправление должно быть постоянным (код 301), чтобы поисковые системы и браузеры запомнили настройку HTTPS.
